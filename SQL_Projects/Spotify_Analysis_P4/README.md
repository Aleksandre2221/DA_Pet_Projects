## Description
This project focuses on performing **data operations, analysis, and reporting** on Spotify tracks using SQL.  
The goal is to handle real-world tasks such as managing tracks, albums, artists, and analyzing streaming trends, views, likes, and audio features.

## Case Study Questions
The analysis answers key business questions, including:

- Retrieving tracks with more than 1 billion streams  
- Listing all albums with their respective artists  
- Calculating total comments for licensed tracks  
- Finding tracks of type "single"  
- Counting the number of tracks per artist  
- Calculating average danceability per album  
- Identifying top tracks with highest energy  
- Listing official videos with views and likes  
- Calculating total views per album  
- Comparing streams across platforms (Spotify vs YouTube)  
- Finding top 3 most-viewed tracks per artist using window functions  
- Detecting tracks with above-average liveness  
- Calculating energy range per album  
- Filtering tracks with high energy-to-liveness ratio  
- Calculating cumulative likes ordered by views  

## Findings
- **High-Stream Tracks:** Several tracks have crossed 1 billion streams, indicating extreme popularity.  
- **Album Insights:** Artists often release multiple tracks across albums, and album-level analysis shows variations in danceability and energy.  
- **Engagement Metrics:** Official videos with higher views and likes can be identified for marketing insights.  
- **Platform Comparison:** Some tracks perform better on Spotify than YouTube, useful for platform-specific strategies.  
- **Artist Performance:** Window functions reveal the top tracks per artist, helping understand audience preferences.  
- **Audio Features:** Metrics like energy, danceability, and liveness provide insight into track characteristics.  
- **Cumulative Engagement:** Summing likes by views highlights tracks that consistently engage listeners.
